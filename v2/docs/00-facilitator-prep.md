# 🧰 Facilitator Prep — อ่านก่อนวันงาน

> ไฟล์นี้สำหรับทีมจัด ไม่ใช่เด็ก เป้าหมาย: กันจุดที่เคยพังหน้างาน โดยเฉพาะเรื่อง reset บอร์ดและ Wi-Fi คอขวด

---

## ⚠️ เรื่อง Reset บอร์ด — อ่านให้จบก่อนตัดสินใจ

บอร์ดวนมาจากที่อื่น รหัส Linux / ชื่อ / Wi-Fi เดิมไม่รู้ **มี reset 2 ระดับ อย่าสับสน:**

### ระดับเบา — App Lab → Settings (ใช้กับเด็กในห้อง)
- ไม่ต้อง jumper, ไม่ต้องโหลด image, เร็ว
- ลบ config เดิม (Wi-Fi ฯลฯ) แล้วให้ตั้ง Wi-Fi/ชื่อใหม่ผ่าน wizard
- **เหมาะกับ 20 ทีมทำพร้อมกัน** — นี่คือ path ที่ให้เด็กทำเอง

### ระดับ nuclear — Arduino Flasher CLI + jumper (ห้ามให้เด็ก 20 ทีมทำพร้อมกัน)
- ต้อง short 2 ขาด้วย jumper เข้า flash mode + โหลด Debian image **>1GB ต่อเครื่อง** + flash 5–10 นาที
- ได้ factory state สะอาดจริง (รหัสว่าง) แต่ **20 เครื่องโหลด 1GB พร้อมกันผ่าน Wi-Fi เดียว = หายนะ**
- ใช้เฉพาะเครื่องที่ "ตายสนิท App Lab มองไม่เห็น" และให้ **TA ทำข้างห้อง** ไม่ใช่เด็ก

### ❗ จุดที่ต้อง verify บนบอร์ดจริงก่อนวันงาน (สำคัญที่สุด)
App Lab login เข้าบอร์ด **อัตโนมัติ** แม้ไม่รู้รหัส Linux เดิม — แปลว่าเด็กใช้บอร์ดทำ workshop ได้โดยไม่ต้องรู้รหัส **แต่ยังไม่ชัดว่า reset ระดับเบาลบ "รหัส Linux เดิม" ออกด้วยหรือเปล่า** (เอกสารบอกว่าจะได้รหัสว่างจริงต้อง reflash เต็ม)

ก่อนวันงาน **เอาบอร์ดตัวอย่าง 2 ตัวมาลอง:**
1. App Lab → Settings → reset config → ผ่าน wizard ตั้งรหัส/ชื่อใหม่ได้ไหม
2. หลัง reset ลอง SSH ด้วยรหัสที่เพิ่งตั้ง — เข้าได้ไหม
   - **เข้าได้** → ระดับเบาพอ ให้เด็กทำเองได้เลย
   - **เข้าไม่ได้ (รหัสเก่ายังค้าง)** → ต้อง pre-reflash บอร์ดทั้งหมดคืนก่อนวันงาน (ทีมจัดทำคืนก่อน ไม่ใช่เด็กทำหน้างาน)

> เหตุผลที่ต้องสน SSH/รหัส: Day 2 ถ้าเด็กต้อง sudo / SSH / ลง package เพิ่ม จะต้องใช้รหัสที่รู้ ตั้งรหัสกลางที่จำง่ายตอน first boot (เช่น `team-XX`) แล้วจดไว้

- [ ] Arduino UNO Q + USB-C cable + power adapter (5V/3A)
- [ ] Modulino kit (อย่างน้อย Movement + Pixels + Buzzer) + Qwiic cable ≥3 เส้น
- [ ] **Powered USB hub** (มีไฟเลี้ยง) — จำเป็น เพราะ webcam/mic + power พร้อมกัน
- [ ] USB Webcam
- [ ] USB Microphone
- [ ] Laptop (ลง Arduino App Lab + Edge Impulse CLI ไว้ล่วงหน้า)

## 🖥️ ของกลางในห้อง

- [ ] Wi-Fi ที่ทน 20 ทีม connect + upload data พร้อมกัน (ดูหัวข้อ Wi-Fi ด้านล่าง)
- [ ] Power strip พอสำหรับบอร์ด + laptop ทุกทีม
- [ ] Projector + HDMI
- [ ] โต๊ะ TA + บอร์ดสำรอง ≥2 + hub สำรอง + cable สำรอง

## 📶 Wi-Fi — จุดพังเงียบ ๆ

3 ช่วงที่ Wi-Fi โดนถล่มพร้อมกัน: (1) first boot App Lab "check for updates", (2) ต่อ Edge Impulse, (3) upload data ตอนบ่าย

ลดความเสี่ยง:
- [ ] **Pre-update บอร์ดทุกตัวก่อนวันงาน** (App Lab check-for-updates ให้จบ) เพื่อไม่ให้เด็ก 20 ทีมโหลด update พร้อมกัน
- [ ] แยก SSID/AP สำหรับ workshop อย่าใช้ Wi-Fi รวมของตึก
- [ ] เตรียม mobile hotspot สำรอง 1–2 ตัวให้ทีมที่หลุด
- [ ] 5GHz ดีกว่า 2.4GHz ถ้ามีให้เลือก

## 🆘 ของพังหน้างาน เปิดอะไรก่อน

| อาการ | ทำอะไร |
|---|---|
| บอร์ด App Lab มองไม่เห็น | เปลี่ยน cable/port (USB-C→C ตรง laptop), เลี่ยง hub ตอน setup, รอ ~60 วิ |
| บอร์ดตายสนิท reset เบาไม่ได้ | สลับบอร์ดสำรองให้เด็กก่อน → TA ค่อย CLI flash ตัวเสียข้างห้อง |
| Webcam/Mic ไม่ขึ้น | เช็ก powered hub มีไฟ, `lsusb` / `ls /dev/video*` / `arecord -l` |
| Wi-Fi ช้าทั้งห้อง | ดึงทีมที่ค้างไป hotspot สำรอง |
| Train/upload ช้า | ลดจำนวน sample/class ลงก่อน ให้จบ flow ดีกว่าสมบูรณ์แต่ไม่จบ |
